Introduction ¶

The concept of the absolute value of a real number, and its generalization to the norm if a complex number, is ubiquitous in mathematics and often serves a key role in various proofs. The absolute value of a real number $x$, or the norm of a complex number $x$, is captured with the operation `Abs(x)`, along with a standard axiomatic definition and a variety of related theorems.

Simple Expressions Involving Absolute Value $|a|$ ¶

It is straightforward to construct absolute value expressions. Here are some basic examples of such expressions:

In [2]:
# basic absolute value expression
abs_value_a = Abs(a)

# inside a larger expression

Axioms ¶

Demonstrations ¶

1. Simplifying $|x+1|$ and Deducing Number Sets

Consider the very simple absolute value expression $|x+1|$, where the operand is a sum of a variable and known constant:

In [19]:
demo_expr_01 = Abs(Add(x, one))
Out[19]:
demo_expr_01:

The general expression simplification() method can eliminate the absolute value wrapper under the right conditions — for example, when we know that $x$ is a positive real number:

In [20]:
demo_expr_01.simplification(assumptions=[InSet(x, RealsPos)])
Out[20]:
⊢ 

We can deduce that the absolute value expression is in a specific number set (given the right assumptions):

In [21]:
demo_expr_01.deduceInNumberSet(RealsNonNeg, assumptions=[InSet(x, Reals)])
Out[21]:
⊢ 

Now when $x \in \mathbb{R}^{+}$, we should have $|x+1|>1$ and thus $|x+1| \in \mathbb{R}^{+}$. Interestingly, this requires a little more work before Prove-It agrees:

In [22]:
from proveit import InstantiationFailure
try:
    demo_expr_01.deduceInNumberSet(RealsPos, assumptions=[InSet(x, RealsPos)])
except InstantiationFailure as e:
    print("SpecializationFailure: " + str(e))
    

SpecializationFailure: Proof step failed assuming {x in RealsPos}:
Attempting to instantiate |- forall_{a in Complexes | a != 0} (|a| in RealsPos) with {a: x + 1}:
Unsatisfied condition: (x + 1) != 0

We could include the information the system seems to want, i.e., that $x+1 \ne 0$, and Prove-It is happy:

In [23]:
demo_expr_01.deduceInNumberSet(
    RealsPos,
    assumptions=[InSet(x, RealsPos), NotEquals(Add(x, one), zero)])
Out[23]:
⊢ 

But that seems a little clunky, with that additional (and redundant) assumption appearing in the result. Instead we can nudge Prove-It by first having it prove that the operand $x+1$ is itself a positive real value, and then it has no problem proving that the absolute value expression $|x+1|$ itself is also a positive real value:

In [24]:
InSet(Add(x, one), RealsPos).prove(assumptions=[InSet(x, RealsPos)])

|- forall_{i in Naturals, j in Naturals} [forall_{a_{1} in RealsNonNeg, ..a_{_a} in RealsNonNeg.., a_{i} in RealsNonNeg, b in RealsNonNeg, c_{1} in RealsNonNeg, ..c_{_a} in RealsNonNeg.., c_{j} in RealsNonNeg | b > 0} ((a_{1} + ..a_{_a}.. + a_{i} + b + c_{1} + ..c_{_a}.. + c_{j}) in RealsPos)] {i: 1, j: 0, a: (x,), b: 1, c: ()}
Out[24]:
⊢ 
In [25]:

2. Simplifying $|a+b+2+3|$ and Deducing Numbers Sets

Now consider a simple absolute value expression, where the operand is a sum of variables and known constants: $|a+b+2+3|$

In [26]:
demo_expr_02 = Abs(Add(a, b, two, three))
Out[26]:
demo_expr_02:

We can obtain simplification equivalences, depending on our assumptions about the nature of the variables $a$ and $b$. If $a$ and $b$ are complex or real, for example, Prove-It simplifies the $2+3$ portion to $5$ and keeps the absolute value:

In [27]:
# variables a and b are complex, notated as a tuple
demo_expr_02.simplification(assumptions=[InSet(a, Complexes), InSet(b, Complexes)])
Out[27]:
⊢ 
In [28]:
# variables a and b are real, notated individually
demo_expr_02.simplification(assumptions=[InSet(a, Reals), InSet(b, Reals)])
Out[28]:
⊢ 

We can simplify further if we know that $a$ and $b$ are non-negative reals, with `simplification()` combining the numerical constants *and* removing the absolute value:

In [29]:
demo_expr_02.simplification(
    assumptions=[InSet(a, RealsNonNeg), InSet(b, RealsNonNeg)])
Out[29]:
⊢ 

Similarly, given the assumption that $a$ and $b$ are non-negative reals, we can explicitly deduce that the entire expression is also a non-negative real:

In [30]:
demo_expr_02.deduceInNumberSet(
        RealsNonNeg,
        assumptions=[InSet(a, RealsNonNeg), InSet(b, RealsNonNeg)])
Out[30]:
⊢ 

In fact, as long as we explicitly establish that $a$ and $b$ each belong to some defined subset of the complex numbers, the `deduceInNumberSet()` will be able to deduce that our absolute value expression is an element of the non-negative reals:

In [31]:
demo_expr_02.deduceInNumberSet(
        RealsNonNeg,
        assumptions=[InSet(a, Complexes), InSet(b, RealsNeg)])
Out[31]:
⊢ 

If we want instead to derive the fact that $|a+b+2+3|\ge 0$, it turns out we have to work a little harder, with the automation not accomplishing everything for us:

In [32]:
# |x| ≥ 0 not derived automatically
from proveit import ProofFailure
try:
    GreaterEq(demo_expr_02, zero).prove(
        assumptions=[InSet(a, Complexes), InSet(b, RealsNeg)])
except ProofFailure as e:
    print("Without a little more help, Prove-It returns an error: {}".format(e))

Without a little more help, Prove-It returns an error: Unable to prove |a + b + 2 + 3| >= 0 assuming {a in Complexes, b in RealsNeg}:
Insufficient known transitive relations to sort the remaining items: {0, |a + b + 2 + 3|}

So we pull in a related theorem, essentially serving as a definition for the set of non-negative reals, $R^{\ge 0}$:

In [33]:
from proveit.number.sets.real._theorems_ import inRealsNonNeg_iff_non_negative
inRealsNonNeg_iff_non_negative
Out[33]:
⊢ 

Then we prove that the absolute value expression in question is real (so that we can specialize the theorem above):

In [34]:
demo_expr_02.deduceInNumberSet(
        Reals,
        assumptions=[InSet(a, Complexes), InSet(b, RealsNeg)])
Out[34]:
⊢ 

And perform the specialization:

In [35]:
inRealsNonNeg_iff_nonNegative_spec = inRealsNonNeg_iff_non_negative.specialize(
        {a:demo_expr_02}, assumptions=[InSet(a, Complexes), InSet(b, RealsNeg)])
Out[35]:
inRealsNonNeg_iff_nonNegative_spec: ⊢ 

Then derive the right-hand side of that logical equivalence (which is possible because we earlier proved the left-hand side of that logical equivalence under the same assumptions):

In [36]:
demo_expr_02_ge_zero = inRealsNonNeg_iff_nonNegative_spec.deriveRight()
Out[36]:
demo_expr_02_ge_zero: ⊢
